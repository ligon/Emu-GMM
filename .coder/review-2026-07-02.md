## Criticals (verified by orchestrator)

### C1. Scalar psi silently broadcasts to (N,N) garbage moments
`src/emu_gmm/measures/empirical.py:134-137, 297-303`
A user psi returning a 0-d scalar per observation (`return beta*r*ratio - 1.0`)
gives `vmap` output shape `(N,)`. `_check_mask_psi_compatibility` explicitly
defers on non-2D shapes, but JAX raises no error: `jnp.where((N,1) mask, (N,)
psi, 0.0)` broadcasts to `(N,N)`, so `expectation` returns an `(N,)` vector of
per-observation residuals as "moments". For K=1 the whole pipeline runs and
reports statistically meaningless results with no error; for K≥2 it crashes
late with a misleading message. This is the documented Hansen-Singleton recipe
(`from_nan_aware(x, M=1)`) one `jnp.array([...])` away from every user.
**Fix**: raise loudly on 1-D psi batch (0-d scalar return) with a pointer.

### C2. `moment_wild_bootstrap` fabricates p = 0.0 under non-PD V
`src/emu_gmm/inference/wild_bootstrap.py:334-343, 367`
With `V=None` the **raw, unregularised** covariance goes straight to Cholesky
(the module has no `regularization` parameter, unlike `k_statistic`/`j_test`).
`cho.cholesky` returns NaN on non-PD input; `J_observed` becomes NaN; every
`J_boot >= NaN` is False; `p_value = mean(...) = 0.0` — a clean-looking hard
rejection. Few-cluster / barely-indefinite V (the exact #111 class) triggers
it; commitment 3's fix lives in `DiagonalTikhonov`, which this path never
invokes. **Fix**: add `regularization=DiagonalTikhonov()` parameter applied on
the internally-computed V, and surface a NaN `J_observed` as NaN p-value.

## Majors

### M1. `DiagonalTikhonov.apply` silently returns an infeasible V* (verified)
`src/emu_gmm/regularization.py:261-294`
The bisection *trusts* `hi = _TAU_MAX` to be feasible and never re-verifies the
returned `V_star`, contradicting its own "guaranteed positive-definite"
docstring. A moment coordinate with `V_jj = 0` (constant psi_j; an indicator
that never fires in a replicate; **any all-zero mask column, since N_j = 0 ⇒
row/col j of V is exactly 0**) can never be repaired by the multiplicative
ridge `(1+tau)·0 = 0`, so `apply` returns a singular V* with tau=1000 and the
downstream Cholesky NaNs the entire fit silently (`_internal/cholesky.py`
returns NaN by design). Three reviewers independently converged on this chain:
- estimator core: no final `_feasible(V_star)` check;
- measures: the `N_j = 0` end-to-end silent-NaN chain (no guard anywhere);
- numerics: `ridge_inverse` (`numerics/ridge_inverse.py:118-135`) returns an
  all-NaN inverse with a no-flag info dict in the same scenario, shipped to
  K-Aggregators.
**Fix direction**: final feasibility check in `apply`; loud (or flagged)
behavior on saturation; an `N_j == 0` guard at the estimator boundary;
`saturated` flag in `ridge_inverse` info. Also fix the `cholesky.py` module
docstring, which claims it "raises a clear error" (it does the opposite).

### M2. Linear solver's traced branch reports every fallback as converged (verified)
`src/emu_gmm/optimizer.py:601-620` + `estimator.py:1195-1197`
Under jit/vmap, `_fallback_branch` returns a 3-tuple that discards
`finfo.done`, and the rebuilt info hardcodes `status="converged"` with
`done=None` — so the estimator's status-string fallback reports **every**
non-converged fallback replicate as converged (the #78 hazard the `done`
channel was added to kill). Monte Carlo over a nonlinear residual with
`linear_solver()` quietly counts max-steps stalls as successes.
**Fix**: thread `done` through both `lax.cond` branches.

### M3. `MCRecords.extra` (#183 per-draw statistics) silently dropped by persistence (verified)
`src/emu_gmm/persistence.py:335-358, 558-577`
`extra` was added in 495686d *after* law persistence landed in 7502787;
`_empirical_state` writes only `_FITRECORD_ARRAY_FIELDS` + key/coupling, and
reload rebuilds `MCRecords` without `extra`. A consumer's custom per-draw
statistics vanish on save/load round-trip with no error. No test covers it.
**Fix**: persist `extra` under a dedicated key namespace + round-trip test.

### M4. `Interval` leaves silently get the Euclidean retraction in both solvers (verified)
`riemannian_lm.py:350-372`, `riemannian_tr.py:184-218`
Scalar-leaf dispatch tests `_is_positive` only; `Interval` (top-level exported,
blessed as a leaf by tests, persisted by law_state) falls to `p0 + d0` with the
identity metric — its retraction/metric are never called, so accepted steps can
violate the bounds it exists to enforce. No test runs `estimate()` with an
Interval leaf. **Fix**: dispatch scalar leaves through `manifold.retraction` /
`inner_product` generically (Euclidean fast-path preserved bitwise).

### M5. `ParameterSpace` field without `on(...)` is silently frozen (verified)
`src/emu_gmm/parameter_space.py:162-165, 202-205`
`__init_subclass__` collects only `_FieldSpec` values and rewrites
`__annotations__` wholesale; a field declared `mu: Array = jnp.zeros(3)`
survives as a plain class attribute — `theta.mu` still resolves, the model
runs, but the "parameter" is held constant during estimation with no warning.
**Fix**: raise on leftover non-`on()` array-valued class attributes/annotations.

### M6. `from_nan_aware(M=...)` inf/NaN disagreement re-creates the NaN-gradient trap (reported, high confidence)
`empirical.py:757 vs 771` + `_internal/nan_safety.py:115`
The `M=` branch masks rows by `isfinite` but the sentinel scrubber replaces
only `isnan` cells — a ±inf cell is masked out of every moment yet survives
into stored `x`, and `psi(inf)` NaNs poison reverse-mode AD through the masked
`where` (the precise hazard this machinery documents). Legacy `M=None` branch
uses `isnan` — the two branches of one constructor disagree on inf semantics.
**Fix**: scrub by `isfinite` consistently (`safe_x_for_psi`, `column_mean_sentinel`).

### M7. `from_arrays` skips all input validation (reported, high confidence)
`empirical.py:822-835`
Unlike `from_pandas`/`from_nan_aware`, `from_arrays` never calls
`_assert_finite_weights` (one NaN weight silently NaNs every moment and N_j),
nor validates weights length/mask shape, nor checks the docstring's own
"asserts complete data" claim. **Fix**: apply the same validation helpers.

### M8. `cluster_bootstrap` refit loop leaks JAX caches (verified pattern)
`inference/cluster_bootstrap.py:449-500`
Per-replicate bare `estimate()` with data-dependent shapes and no
`jax.clear_caches()` — the documented ~14 MB/call OOM pattern that
`profiled_k_confidence_set` (every grid point) and `replicate` (every 25 reps)
both already mitigate. **Fix**: clear every N replicates.

### M9. NamedArray return silently discards a user's `moment_names` kwarg (deliberate but wrong-way)
`_internal/labels.py:247-257`
The NamedArray branch outranks the explicit kwarg but carries zero label
information (haliax Axis has no per-coordinate names) — the user's real labels
are replaced by `m_0, m_1, ...`, and a test pins this precedence. Drifts from
CLAUDE.md commitment 6's rationale. **Owner call** (test pins it): fall through
to the kwarg when the axis carries no coordinate labels.

### M10. `AnalyticalMeasure` AD fallback still uses v1 scalar-only flatten (loud, missed migration)
`measures/analytical.py:99-105`
`empirical`/`synthetic` were migrated to `flatten_params_for_ad` (#41);
`analytical` was missed — manifold parameter trees crash with a stale v1 error,
breaking Measure × manifold orthogonality. **Fix**: same migration.

## Statistical-design items (theory-flagged; need owner judgment / MC evidence, not mechanical fixes)

- **S1. Wild bootstrap ignores the estimation effect** (`wild_bootstrap.py`):
  documented for evaluation at θ̂, but sign-flipped draws are uncentred and
  unprojected ⇒ J_boot targets ~χ²_M while J_observed(θ̂) ~ χ²_{M−K} —
  conservative p-values in over-identified models. Calibration tests only cover
  fixed true θ0 with M=1. Consider recentring/projection or documenting the
  θ0-only contract.
- **S2. Adaptive-bootstrap docs recommend un-recentred `J_boot`**
  (`adaptive.py:23-25`): pairs-bootstrap J without Hall–Horowitz recentring is
  an invalid over-id test; the docstring recipe hands users exactly that.
- **S3. `profiled_k_confidence_set` never checks `inner.converged`**
  (`confidence_set.py:689-716`): a stalled nuisance concentration yields a
  finite-but-wrong K, indistinguishable from a good grid point (spurious
  rejection ⇒ under-coverage). Cheap fix: surface/flag non-converged points.
- **S4. Subvector χ²_{d_I} validity requires CU nuisance concentration**
  (`confidence_set.py:557,607-609`): `nuisance_weighting=Fixed(...)` silently
  invalidates the reference distribution; needs a caveat or a warning.
- **S5. `J_pvalue` emitted under any weighting** (`estimator.py:879-892`):
  under `Identity`/mis-scaled `Fixed` the χ²_{M−K} reference is wrong;
  diagnostics docstring says "treat as unavailable" but a number is returned.
- **S6. `result.V_X` stores the regularised V\*** (`types.py:558`,
  `estimator.py:1101`) while docs describe the raw pairwise-overlap estimator —
  plausibly deliberate (V* is what J/Σθ used); the field doc should say so.
