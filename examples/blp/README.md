# BLP demand estimation: emu-gmm vs pyblp

A validation study: estimate the Berry–Levinsohn–Pakes random-coefficients
logit demand model with `emu-gmm` and compare, quantity by quantity, against
`pyblp` on its Nevo "fake cereal" tutorial data. The point is correctness
confidence — reproducing an independent, well-tested implementation on a real
structural GMM problem.

**Read `REPORT.md` for the findings.** Headline: emu reproduces pyblp's point
estimates, GMM objective, J-statistic, and standard errors to 6–8 significant
digits across logit, RC-logit, and RC-logit-with-demographics, including from a
cold start.

## Files

| file | role |
|---|---|
| `build_reference.py` | solve the 3 specs in pyblp; save its estimates, optimal weight matrices, and extracted internal arrays to `reference/` |
| `blp_model.py` | JAX-native BLP: the Berry contraction as an implicitly-differentiated (`custom_jvp`) fixed point, and the per-product-market `psi` for emu |
| `blp_data.py` | load the pyblp reference arrays into emu-side bundles |
| `run_emu.py` | estimate with `emu_gmm.estimate` and print the comparison |
| `sanity_check.py` | model de-risking: contraction vs pyblp delta, IFT derivative vs finite differences, moment/objective scaling |

## Run

Requires `pyblp` in the poetry venv (`pip install pyblp`). CPU affinity per the
repo's JIT-mmap note:

```bash
XLA_FLAGS=--xla_force_host_platform_device_count=1 taskset -c 0-3 \
    .venv/bin/python examples/blp/build_reference.py        # pyblp reference (~45s)
cd examples/blp && XLA_FLAGS=--xla_force_host_platform_device_count=1 taskset -c 0-3 \
    ../../.venv/bin/python run_emu.py all                   # emu + comparison
```

`reference/` is generated by `build_reference.py` and not committed.
