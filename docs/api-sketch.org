* Section 1: Scope and Conventions

This sketch describes the v1 surface of =emu-gmm=: the minimal API needed to run a structural-GMM estimation end-to-end --- against a synthetic generator, a closed-form analytical population, or real empirical data --- through a single operator interface.

** In Scope (v1)
- =StructuralModel= as a callable; user-supplied plain function or =@jdc.pytree_dataclass=.
- =SyntheticMeasure= over a frozen PRNG-key batch with a user-supplied sampler. Sampler may or may not depend on $\theta$.
- =EmpiricalMeasure= over a sample with per-coordinate missingness mask and pweighted observations.
- =AnalyticalMeasure= for closed-form populations (no quadrature library yet --- closed-form integrators only).
- =CovarianceStrategy= implementations: =SyntheticCovariance=, =IIDCovariance=, =ClusteredCovariance=, =AnalyticalCovariance=.
- =WeightingStrategy=: =Identity=, =Fixed=, =ContinuouslyUpdated= (default). =Iterated= deferred --- CU subsumes it.
- =RegularizationStrategy=: =DiagonalTikhonov= only.
- =Optimizer= protocol with =optimistix_lm()= default; =scipy_lm()= adapter for the non-JIT-pure fallback.
- Stateless =estimate(...)= entry point.
- =EstimationResult= with provenance and a v1 =Diagnostics= dataclass.

** Deferred to v2+
- *SMM matching* --- combining =EmpiricalMeasure= and =SyntheticMeasure= in one moment vector, including the $(1+1/S)$ asymptotic-variance inflation. v1 supports each measure independently but does not combine them.
- =StratifiedCovariance=, =ReplicateWeightCovariance=.
- =Iterated= weighting (subsumed by CU for v1 purposes).
- =EigenvalueFloor= and =NearestPSD= regularisers.
- The full identification-diagnostics taxonomy (=design.org= Section 5.4) as machine-readable structured reports.
- Quadrature-based analytical measures.
- Replicate-weight statistics (BRR, jackknife, bootstrap).

** Dependencies
- =jax= (>= 0.4.something current)
- =haliax= (named axes / PyTree-friendly tensors)
- =jaxtyping= (compile-time shape annotations)
- =jax_dataclasses= (pytree-dataclass decorator, exposed as =jdc=)
- =optimistix= (NLLS solver; LM and trust-region)
- =numpy=, =scipy= (numpy for the scipy fallback, scipy.stats for J-test p-values)

No =pandas= /inside/ =jit= / =vmap= / AD boundaries. But at the input boundary, =estimate(...)= accepts any of: plain JAX arrays, =haliax.NamedArray=, or =pandas.DataFrame= / =pandas.Series=. The framework's polymorphic-input adapter normalises to JAX arrays with axis-label tracking. At the output boundary, =EstimationResult= always carries labelled =haliax.NamedArray= for matrix-valued fields (=Sigma_theta=, =V_X=, per-moment =Diagnostics=), with a =result.to_pandas()= convenience converter for pandas-centric workflows. The user's parameter dataclass is preserved on output so =result.theta_hat.beta= continues to work.

* Section 3: Core Types

** StructuralModel
A =StructuralModel= is anything callable matching the signature
\[ \texttt{psi}: (x, \theta) \to \mathbb{R}^M \]
where $x$ is a single observation (1-D array of length $D$) and $\theta$ is the parameter vector.

The framework supports two authoring styles:

*** Plain function (default)
#+BEGIN_SRC python
def euler_residual(x, theta):
    c, c_next, r = x
    beta, gamma = theta.beta, theta.gamma
    return jnp.array([beta * (c_next / c) ** (-gamma) * (1 + r) - 1.0])
#+END_SRC

*** Dataclass with hyperparameters (opt-in)
#+BEGIN_SRC python
@jdc.pytree_dataclass
class EulerModel:
    risk_free_rate: float = jdc.static_field(default=0.02)

    def __call__(self, x, theta):
        c, c_next, r = x
        adj_r = r + self.risk_free_rate
        return jnp.array([theta.beta * (c_next / c) ** (-theta.gamma) * (1 + adj_r) - 1.0])
#+END_SRC

The framework duck-types: both forms work. Dataclass authoring is the right tool when the model has calibration constants or auxiliary objects that need to ride through =jit=/=vmap=.

Optionally, the user can provide an analytical gradient via a paired callable (e.g., =psi_grad(x, theta)= returning the $M \times K$ Jacobian); the framework falls through to =jax.jacfwd= when not supplied.

** Parameters
Parameters are a user-defined =@jdc.pytree_dataclass=:
#+BEGIN_SRC python
@jdc.pytree_dataclass
class EulerParams:
    beta: float
    gamma: float
#+END_SRC

The framework converts this to a flat JAX array internally (via =jax.tree_util=) for AD operations, and reconstructs the same dataclass on output. Field order is preserved as the canonical parameter order.

** Label Conventions
The framework accepts labelled inputs and produces labelled outputs.  Labels for each axis come from a defined source:

- *Params*: parameter dataclass field names. Free from the user's =@jdc.pytree_dataclass=.
- *Moments*: in order of preference:
  1. If the =StructuralModel= callable returns a =haliax.NamedArray= with a =Moments= axis, those coordinates are used.
  2. Otherwise, if =estimate(...)= is called with an explicit =moment_names=("euler", "excess_return", ...)= kwarg, those are used.
  3. Otherwise, positional fallback: =["m_0", "m_1", ...]=.
- *Variables* (the columns of an observation $x$): for =EmpiricalMeasure=, taken from =pandas.DataFrame= column names if =x= is passed as a DataFrame, or from a =haliax.NamedArray='s coordinates; positional otherwise.
- *Obs*: positional. Index labels are preserved if the user passes a DataFrame with a named index, but the framework does not rely on observation labels for any computation.

The labels are tracked in a small internal =LabelContext= that rides through the estimation pipeline as a static argument (not traced). All matrix-valued outputs in =EstimationResult= and =Diagnostics= are =haliax.NamedArray= instances carrying the appropriate coordinates.

*** AnalyticalMeasure
#+BEGIN_SRC python
@jdc.pytree_dataclass
class AnalyticalMeasure:
    expectation_fn: Callable[[StructuralModel, ParamsLike], Float[Array, "M"]]
    jacobian_fn: Optional[Callable[[StructuralModel, ParamsLike], Float[Array, "M K"]]] = None

*** SyntheticMeasure
#+BEGIN_SRC python
@jdc.pytree_dataclass
class SyntheticMeasure:
    """Generator-backed measure. The sampler produces n_sim observations
    from a frozen PRNG key (Common Random Numbers). The sampler may
    depend on theta (genuine SMM-style generator) or ignore it (the
    Euler demo case where the data-generating process is exogenous);
    the framework's AD treats both uniformly."""
    key: jax.Array                                     # frozen PRNGKey
    n_sim: int = jdc.static_field()
    sampler: Callable[[jax.Array, ParamsLike], Float[Array, "n_sim D"]]

    # expectation: vmap psi over the n_sim simulated draws and average.
    # jacobian: jax.jacfwd of the expectation; AD threads through both
    # the sampler (if it depends on theta) and psi.
#+END_SRC

In v1 the =SyntheticMeasure= is the natural self-contained demo path: no external data file is required, the moment estimator is deterministic in $\theta$ (CRN), and the same operator interface (=expectation=, =jacobian=) drives the estimator. Combining a =SyntheticMeasure= with an =EmpiricalMeasure= for SMM matching --- with the $(1+1/S)$ inflation factor in the inference engine --- is v2.

** CovarianceStrategy (Protocol)
#+BEGIN_SRC python
class CovarianceStrategy(Protocol):
    def covariance(
        self, psi: StructuralModel, theta: ParamsLike, measure: Measure

        """Construct V_mu(theta) for the given (model, measure)."""
#+END_SRC

Concrete implementations:

*** IIDCovariance (empirical, pairwise overlap)
#+BEGIN_SRC python
@jdc.pytree_dataclass
class IIDCovariance:
    """Pairwise-overlap iid variance.

    [V_X(theta)]_{jk} = (1 / N_j N_k) sum_i d_ij d_ik w_i^2 psi_j psi_k
    """
    # No state; the variance is computed from the measure at evaluation time.
#+END_SRC

*** ClusteredCovariance (empirical, cluster totals)
#+BEGIN_SRC python
@jdc.pytree_dataclass
class ClusteredCovariance:
    """Cluster-totals form.

    [V_X(theta)]_{jk} = (1 / N_j N_k) sum_c (sum_{i in c} d_ij w_i psi_j) x (...)
    """
    cluster_ids: Float[Array, "N"]
    n_clusters: int = jdc.static_field()
#+END_SRC

*** AnalyticalCovariance (closed-form)
#+BEGIN_SRC python
@jdc.pytree_dataclass
class AnalyticalCovariance:
    """User-supplied closed-form V(theta) under the population."""
    covariance_fn: Callable[[StructuralModel, ParamsLike], Float[Array, "M M"]]
#+END_SRC

*** SyntheticCovariance (Monte Carlo)
#+BEGIN_SRC python
@jdc.pytree_dataclass
class SyntheticCovariance:
    """Monte Carlo variance of the simulated moment estimator.

        V_sim(theta) = (1/n_sim) * (E_z[psi psi'] - E_z[psi] E_z[psi]')
                       evaluated over the frozen draws.

    Scales consistently with the other strategies as the variance of the
    sample moment estimator (not the per-draw variance)."""
    # No state; the variance is constructed from the measure's draws and
    # the StructuralModel at evaluation time.
#+END_SRC

The $(1+1/S)$ inflation that distinguishes SMM from GMM appears only when a =SyntheticContext= is /combined/ with an =EmpiricalContext= in a matching estimator (v2). Pure-synthetic v1 estimation has no inflation factor.

** WeightingStrategy
#+BEGIN_SRC python
class WeightingStrategy(Protocol):
    def whitening_residual(
        self, m: Float[Array, "M"], V: Float[Array, "M M"], theta: ParamsLike
    ) -> Float[Array, "M"]:
        """Return y = L^-1 m where V = L L'. For Fixed/Identity strategies V
        is held constant; for CU the residual carries dependence on theta
        through L for JAX AD to pick up."""
#+END_SRC

Implementations:
- =Identity= --- $\Lambda \equiv I_M$.
- =Fixed(V0)= --- $L$ from a precomputed $V_0$, frozen.
- =ContinuouslyUpdated()= (default) --- $L(\theta)$ recomputed each call; JAX traces through.

* Section 4: The =estimate()= Entry Point

#+BEGIN_SRC python
def estimate(
    model: StructuralModel,
    measure: Measure,
    *,
    covariance: CovarianceStrategy,
    weighting: WeightingStrategy = ContinuouslyUpdated(),
    regularization: Optional[RegularizationStrategy] = DiagonalTikhonov(),
    optimizer: Optimizer = optimistix_lm(),
    theta_init: ParamsLike,
) -> EstimationResult:
    """
    Estimate theta by minimising Q_mu(theta) = ||L_mu(theta)^-1 m_mu(theta)||^2.

    Steps:
      1. Flatten theta_init to a JAX array (preserving the user's dataclass type).
      2. Build residual_fn(theta_flat) that computes:
           m       = measure.expectation(model, theta)
           V       = covariance.covariance(model, theta, measure)
           V_star  = regularization.apply(V) at anchor (frozen for the run)
           L       = cholesky(V_star)
           y       = forward_solve(L, m)
         The whitening strategy controls whether L depends on theta_flat (CU)
         or is precomputed and frozen (Fixed/Identity).
      3. Hand residual_fn to optimizer; collect (theta_hat_flat, info).
      4. At theta_hat_flat:
           - reconstruct user dataclass type
           - compute G = measure.jacobian(model, theta_hat) (via AD or user-supplied)
           - compute Sigma_theta = (G' V^-1 G)^-1
           - compute J statistic, J_dof, J_pvalue via scipy.stats.chi2
           - collect diagnostics
      5. Return EstimationResult.
    """
#+END_SRC

** Behavioural notes
- All keyword-only after =measure=, to make the call site self-documenting.
- The =CovarianceStrategy= is required; there is no implicit default. The user must opt into a specific design.
- =regularization= can be =None= to disable; the framework will let the Cholesky raise if $V$ is non-PD.
- =theta_init= is the user's parameter dataclass; the framework normalises internally.

* Section 5: Worked Example --- Multi-Asset Euler Equation

The v1 demo is the classical Hansen-Singleton 1982 consumption-based asset-pricing setup: with power utility and $J$ risky assets, the Euler equation for each asset is
\[ \mathbb{E}\!\left[\beta\, (c_{t+1}/c_t)^{-\gamma}\,(1 + r_j)\right] = 1, \]
which gives $J$ moment conditions in $K = 2$ structural parameters $(\beta, \gamma)$. With $J = 3$ assets, $M = 3$ and the system is over-identified ($J_\text{dof} = 1$), so $\Sigma_{\hat\theta}$ is invertible and the J-test is meaningful.

The DGP is engineered so all $J$ Euler conditions vanish exactly at the truth $(\beta_0, \gamma_0) = (0.96, 2.0)$: a single shock $z \sim \mathcal{N}(0,1)$ drives both consumption growth and asset returns, with asset-specific factor loadings $\nu_j$ and idiosyncratic noise so $\operatorname{Var}_{\mu}[\psi]$ is full rank. See =src/emu_gmm/examples/euler.py= for the closed-form derivation and the shared model definitions.

** The Model
#+BEGIN_SRC python
from emu_gmm.examples.euler import EulerParams, euler_residual

# EulerParams = @jdc.pytree_dataclass(beta: float, gamma: float)
# euler_residual(x, theta): returns the J moment residuals
#   psi_j(x, theta) = beta * (c'/c)^{-gamma} * (1 + r_j) - 1
# where x = (c_t, c_{t+1}, r_1, ..., r_J).
#+END_SRC

The next three subsections show the same model estimated via three different =Measure= / =CovarianceStrategy= pairings. The =StructuralModel= itself --- =euler_residual= --- is reused unchanged across all three; only the integration and variance strategies change.

** Variant: SyntheticMeasure (the v1 self-contained demo)
A generator simulates $(c_t, c_{t+1}, r_1, r_2, r_3)$ tuples on demand. No external data file required; Common Random Numbers (frozen =PRNGKey=) makes the objective deterministic in $\theta$.

#+BEGIN_SRC python
import jax
from emu_gmm import estimate
from emu_gmm.measures import SyntheticMeasure
from emu_gmm.covariance import SyntheticCovariance
from emu_gmm.weighting import ContinuouslyUpdated
from emu_gmm.regularization import DiagonalTikhonov
from emu_gmm.optimizer import optimistix_lm
from emu_gmm.examples.euler import (
    EulerParams, euler_residual, euler_sampler_factory,
)

N_SIM = 5000
measure = SyntheticMeasure(
    key=jax.random.PRNGKey(0),
    n_sim=N_SIM,
    sampler=euler_sampler_factory(N_SIM),
)

result = estimate(
    model=euler_residual,
    measure=measure,
    covariance=SyntheticCovariance(),
    weighting=ContinuouslyUpdated(),
    regularization=DiagonalTikhonov(),
    optimizer=optimistix_lm(rtol=1e-8),
    theta_init=EulerParams(beta=0.9, gamma=1.0),
)

print(f"beta = {result.theta_hat.beta:.4f}    (truth 0.96)")
print(f"gamma = {result.theta_hat.gamma:.4f}   (truth 2.0)")
print(f"J-stat = {result.J_stat:.3f}  (dof={result.J_dof}, p={result.J_pvalue:.3f})")
print(f"Converged: {result.converged} in {result.iterations} iterations")
#+END_SRC

Recovery in practice (N_SIM=5000, seed 0): $\hat\beta \approx 0.962$, $\hat\gamma \approx 2.08$, both within Monte Carlo error of the truth. See =tests/test_estimator.py= for the acceptance test.

** Variant: AnalyticalMeasure (identification round-trip)
Pair the same model with the closed-form expectation under the DGP. With no Monte Carlo noise, recovery is exact to floating-point tolerance.

#+BEGIN_SRC python
from emu_gmm.measures import AnalyticalMeasure
from emu_gmm.covariance import AnalyticalCovariance
from emu_gmm.examples.euler import euler_analytical_expectation
import jax.numpy as jnp

def identity_covariance(model, theta):
    return jnp.eye(3)   # any PD V suffices for the identification check

result = estimate(
    model=euler_residual,
    measure=AnalyticalMeasure(expectation_fn=euler_analytical_expectation),
    covariance=AnalyticalCovariance(covariance_fn=identity_covariance),
    weighting=ContinuouslyUpdated(),
    optimizer=optimistix_lm(rtol=1e-10, atol=1e-10),
    theta_init=EulerParams(beta=0.8, gamma=1.0),
)
# result.theta_hat = (0.96, 2.0) to FP precision; result.diagnostics.final_objective < 1e-10.
#+END_SRC

See =tests/test_estimator_analytical.py=.

** Variant: EmpiricalMeasure (the real-data path; demo on pre-generated data)
=EmpiricalMeasure= holds fixed observations as plain JAX arrays (or accepts a =pandas.DataFrame= via =EmpiricalMeasure.from_pandas=). Cluster-robust variance via =ClusteredCovariance= needs an extra =cluster_ids= argument; the iid case uses =IIDCovariance=.

#+BEGIN_SRC python
from emu_gmm.measures import EmpiricalMeasure
from emu_gmm.covariance import IIDCovariance
from emu_gmm.examples.euler import euler_data
import jax.numpy as jnp

N = 5000
x = euler_data(seed=0, n=N)              # (N, 5) JAX array
measure = EmpiricalMeasure(
    x=x,
    mask=jnp.ones((N, 3)),
    weights=jnp.ones(N),
)

result = estimate(
    model=euler_residual,
    measure=measure,
    covariance=IIDCovariance(),
    theta_init=EulerParams(beta=0.9, gamma=1.0),
)
#+END_SRC

The framework enables JAX float64 by default (in =src/emu_gmm/__init__.py=) because float32 isn't precise enough for the optimiser's convergence criteria on a typical whitened-residual surface --- optimistix's strict tolerances fall below the float32 noise floor. Users who explicitly want float32 can override after import via =jax.config.update("jax_enable_x64", False)=.

** What this demonstrates
The same =StructuralModel= (=euler_residual=) drives all three variants. Only the =Measure= and =CovarianceStrategy= change. This is the framework's central architectural claim made operational: one operator interface ($\mathbb{E}_\mu$), three implementations, one =estimate(...)= call site. The =EulerParams= dataclass round-trips through the optimisation without the user touching a flat parameter vector.

* Section 6: Open Implementation Questions

These are real questions that need answers before code can be written, not architectural redesigns.

1. *Polymorphic input adapter.* Resolved: the framework accepts plain JAX arrays, =haliax.NamedArray=, and =pandas.DataFrame= / =pandas.Series= at the input boundary; outputs always carry labels via =haliax.NamedArray=, with a =result.to_pandas()= converter for pandas workflows. See the /Label Conventions/ subsection in Section 3 and the =from_pandas= classmethod sketched in =EmpiricalMeasure=. Implementation lives in =_internal/labels.py= (Phase 1 of =docs/implementation-plan.org=). Open sub-question: should moment labels come from a =StructuralModel= returning a =NamedArray=, or from a =moment_names= kwarg on =estimate=, or both? The sketch currently allows both, with the model-return path taking precedence.

2. *Cluster ID representation.* JAX prefers floats. Integer-typed cluster IDs work in =jax.ops.segment_sum= but the static =n_clusters= argument forces recompilation on cardinality changes. Plan: pad cluster IDs to a fixed maximum at construction time and use =n_clusters= as a static upper bound.

3. *Anchor point for $\tau$.* The design says "first-stage $\hat\theta_1$ for Fixed/Iterated; pilot evaluation for CU." For v1 with only Identity/Fixed/CU strategies, the policy reduces to: $\tau$ is selected once at the start of optimisation (using either $\theta_{init}$ for CU, or the Fixed strategy's anchor $V_0$ for Fixed). Hold fixed thereafter. Document explicitly.

4. *J-test under regularisation.* The weighted-$\chi^2$ characterisation in =design.org= Section 5 / =mcar-asymptotics.org= Theorem 6 is exact in the limit but computationally heavy. v1 reports the nominal $\chi^2_{M-K}$ p-value and flags =binding_ridge= when $\tau > \tau_{threshold}$; computing the adjusted p-value is deferred unless the binding flag fires.

5. *Static vs traced fields on dataclasses.* The patterns above use =jdc.static_field()= for hyperparameters (e.g., =n_clusters=, =kappa_target=). Confirm these are appropriately static --- recompilation when they change is intended.

6. *Output dataclass reconstruction.* The framework needs to round-trip the user's parameter dataclass through a flat JAX array. The implementation uses =jax.tree_util.tree_flatten= / =tree_unflatten=, which assumes the dataclass leaves are scalar-like JAX arrays. Document the restriction; provide a clear error message if violated.

7. *Optimistix interaction with `vmap`.* If a user wants to =vmap= a sweep over hyperparameter grids (e.g., fitting the same model to many cuts of the data), the optimiser must be vmappable. =optimistix= supports this. =scipy_lm= does not. Document the constraint when the scipy adapter is used.

8. *Sampler signature for =SyntheticMeasure=.* The signature is =sampler(key, theta) -> (n_sim, D)=, but for $\theta$-independent samplers (the Euler demo case) the user is forced to accept and ignore =theta=. Acceptable boilerplate for v1; revisit if the pattern proves annoying in practice. A polymorphic overload accepting =sampler(key)= would require runtime introspection of the user's callable.
