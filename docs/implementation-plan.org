* Section 1: Scope and Sequencing

This plan operationalises =docs/api-sketch.org= as a sequence of phases. The order is:

1. /Foundations/ --- internal plumbing nothing else can be written without.
2. /Synthetic path/ --- the v1 demo target; first thing that runs end-to-end.
3. /Inner pipeline/ --- weighting, regularisation, Cholesky.
4. /Optimiser/ --- optimistix adapter.
5. /Entry point/ --- =estimate()= ties it together; first acceptance test fires.
6. /Analytical path/ --- closed-form identification round-trip; second acceptance test.
7. /Empirical path/ --- ready for real data; unit-tested against synthetic inputs only.
8. /Polish/ --- public API re-exports, README, pre-commit.

Each phase has dependencies, tasks, and an acceptance criterion. The first end-to-end run is at the end of Phase 5 (synthetic Euler estimation). Real-data acceptance is deferred to whenever data arrives.

Effort estimates are rough: /S/ ≈ 1-2 hours, /M/ ≈ half-day, /L/ ≈ full day. They are calibrated against /Sue + Claude working together/ pair-of-hours, not solo developer time.

* Section 5: Phase 3 --- Inner Pipeline

/Dependencies: Phase 1./
/Acceptance: each strategy independently testable; composition (=residual_fn = weighting o regularization o measure o covariance=) tested in Phase 5./

** TODO src/emu_gmm/weighting.py                                           :M:
Three weighting strategies.

- =Identity= --- whitening returns $m$ unchanged (equivalent to $L = I$).
- =Fixed(V0)= --- pre-cholesky $V_0$ at construction; reuse $L_0$ on every call.
- =ContinuouslyUpdated()= --- recompute Cholesky of $V(\theta)$ at each call; JAX AD traces through.

The =Fixed= constructor takes a $V_0$ array and stores its Cholesky factor, not the original $V$.

/Tests:/ For a known $V$, =Identity= passes through; =Fixed(V)= whitens to identity-covariance on a simulated normal sample; =ContinuouslyUpdated()= passes a smoke test that AD through =whitening_residual= produces non-zero gradients w.r.t. =theta=.

** TODO src/emu_gmm/regularization.py                                      :S:
=DiagonalTikhonov= regulariser.

- =apply(V, anchor=None)=: if =anchor= provided, compute $\tau$ at the anchor's $V$; otherwise compute $\tau$ at the current $V$. Return =(V_star, tau, kappa)=.
- $\tau$ computed via condition-number bisection (target $\kappa \leq 10^6$ by default).
- A static =tau_threshold= field controls when =binding_ridge= flag fires.

/Tests:/ On a $V$ with $\kappa = 10^8$, output has $\kappa \leq \kappa_{\mathrm{target}}$; $\tau$ scale-equivariant under per-moment rescaling; binding flag fires correctly.

* Section 7: Phase 5 --- Entry Point + First End-to-End

/Dependencies: Phases 1, 2, 3, 4./
/Acceptance: The Euler example in =docs/api-sketch.org= Section 5 runs end-to-end and converges./

** TODO src/emu_gmm/diagnostics.py                                         :S:
=Diagnostics= constructor helpers and logging hooks.

- =collect_diagnostics(state)=: builds a =Diagnostics= from intermediate state computed during =estimate=.
- A simple =log_to_stdout()= hook that prints regularisation $\tau$, condition number, and per-step objective.

/Tests:/ =Diagnostics= is a valid PyTree; =collect_diagnostics= populates all fields without =NaN=.

** TODO src/emu_gmm/estimator.py                                           :L:
The =estimate()= entry point.

- Flatten =theta_init= to a flat JAX array via =_internal/params=.
- Build a closure =residual_fn(theta_flat)= that:
  1. Unflattens =theta_flat= back to the user's params dataclass.
  2. Calls =measure.expectation(model, theta)= → $m$.
  3. Calls =covariance.covariance(model, theta, measure)= → $V$.
  4. Calls =regularization.apply(V)= → $V^\star$ (anchor-once-then-freeze policy).
  5. Calls =weighting.whitening_residual(m, V_star, theta)= → $y$.
  6. Returns $y$ as the NLLS residual.
- Hand =residual_fn= and =theta_init_flat= to =optimizer=.
- At =theta_hat_flat=:
  - Unflatten to the user's dataclass.
  - Compute =G = measure.jacobian(model, theta_hat)= via AD.
  - Compute =Sigma_theta = inv(G' V^{-1} G)= via Cholesky.
  - Compute =J_stat = ||y_hat||^2=; =J_dof = M - K=; =J_pvalue= via =scipy.stats.chi2.sf=.
  - Collect diagnostics; build =EstimationResult=.

The anchor-once-then-freeze policy is implemented by computing $\tau$ at $\theta_{\mathrm{init}}$ and treating it as a non-traced static, so the regularised $V^\star$ at later $\theta$ uses the same $\tau$.

/Acceptance test:/ The Euler example in =docs/api-sketch.org= Section 5 runs and converges to a point on the one-dimensional identified locus. =Diagnostics= report sensible values.

* Section 9: Phase 7 --- Empirical Path (Ready for Real Data)

/Dependencies: Phase 5./
/Acceptance: Unit tests against synthetic data fed via =EmpiricalMeasure= pass; no real-data acceptance test (waiting for data)./

** TODO src/emu_gmm/measures/empirical.py                                  :M:
=EmpiricalMeasure= dataclass.

- Fields: =x= (=(N, D)= array), =mask= (=(N, M)= 0/1), =weights= (=(N,)=).
- =expectation(psi, theta)=: =vmap= =psi= across =N=, apply =mask= and =weights=, normalise by per-coordinate $N_j = \sum_i d_{ij} w_i$.
- =jacobian(psi, theta)=: same shape, but using =jacfwd= of =psi=.

/Tests:/ Coordinate-wise mean on uniform data matches simple =np.mean=; missingness handled correctly per coordinate; weights of all-ones equivalent to unweighted.

** TODO src/emu_gmm/covariance/iid.py                                      :M:
=IIDCovariance= pairwise-overlap form.

- =covariance(psi, theta, measure)=: =einsum= over the pairwise-overlap mask × $\psi_j \psi_k$ × $w_i^2$.

/Tests:/ Special case with full mask and uniform weights matches =numpy.cov= on the $\psi$ matrix; pairwise-overlap test with a strict-subset missingness pattern matches the analytic formula on a 3x3 toy.

** TODO src/emu_gmm/covariance/clustered.py                                :M:
=ClusteredCovariance= cluster-totals form.

- Resolve Section 6 question 2 (cluster ID representation) first: pad cluster IDs to a static maximum and use =jax.ops.segment_sum=.
- Fields: =cluster_ids=, =n_clusters= (static).
- =covariance(psi, theta, measure)=: segment_sum within cluster, outer product, sum over clusters, divide by $N_j N_k$.

/Tests:/ With each cluster of size 1, reduces to =IIDCovariance=; on a toy two-cluster example, matches the analytic cluster-robust formula.

* Section 13: Out of Scope for This Plan

Things the API sketch flags but this plan does not schedule:

- /v2 SMM matching/ --- combined empirical + synthetic moment vectors, $(1+1/S)$ inflation, gradient threading through both contexts.
- /v2 design-aware variance/ --- stratified, replicate-weight (BRR, jackknife, bootstrap), finite-population corrections.
- /v2 regularisation/ --- eigenvalue-floor, Higham nearest-PSD.
- /v2 diagnostics taxonomy/ --- the structured identification-failure reports per =design.org= Section 5.4.
- /v2 quadrature/ --- Gauss-Hermite / Gauss-Legendre engines for =AnalyticalMeasure=.
- /v2 named-tensor surface/ --- Haliax NamedArrays at the user-facing boundary.

These return to the table after v1 ships and a few real estimations have surfaced what actually needs to come next.
