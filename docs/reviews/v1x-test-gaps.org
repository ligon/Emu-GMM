** [HIGH] Binding ridge: =J_pvalue= uses the nominal chi^2, not the regularization-adjusted critical value
   - Confirmed gap. Probe runs a problem where ~binding_ridge=True~
     and ~tau_realised ~ 0.002~ but =J_pvalue= is computed via
     ~scipy.stats.chi2.sf(J_stat, J_dof)~ unconditionally. Per
     =docs/design.org= Section 5: "the inference engine reports both
     the nominal chi^2 critical value and a regularization-adjusted
     critical value when tau exceeds a configurable threshold". The
     adjusted weighted-chi^2 distribution (see
     =docs/mcar-asymptotics.org= line 142) is not computed anywhere.
   - Sketch:
     #+begin_src python
     # tests/test_estimator.py
     def test_J_pvalue_adjusted_when_ridge_binding():
         r = estimate(..., regularization=DiagonalTikhonov(
                              kappa_target=1e3, tau_threshold=1e-6))
         assert r.diagnostics.binding_ridge is True
         # Either a separate adjusted field, or J_pvalue must reflect
         # the weighted-chi^2 mixture, not chi^2_{M-K}.
         assert hasattr(r, "J_pvalue_adjusted") or \
                r.J_pvalue != scipy.stats.chi2.sf(r.J_stat, r.J_dof)
     #+end_src

** [MED] =ClusteredCovariance= with =n_clusters = 1=
   - Confirmed gap. Probe runs but the resulting =V= has
     ~kappa ~ 1e17~ (numerically singular: rank one by construction
     because the only "cluster" is the entire sample). The estimator
     then leans entirely on =DiagonalTikhonov= to recover PD-ness. No
     test exercises this; design.org documents the case implicitly as
     a degenerate fallback, but the practical impact is that a user
     who mis-codes ~cluster_ids=jnp.zeros(N)~ silently gets a
     pathologically conditioned weight matrix and tiny standard
     errors.
   - Sketch:
     #+begin_src python
     # tests/covariance/test_clustered.py
     def test_single_cluster_is_rank_one_and_warned_or_errored():
         # n_clusters=1 reduces V to a single outer product; either
         # raise a clear error or document the rank-one nature.
         cov = ClusteredCovariance(cluster_ids=jnp.zeros(N), n_clusters=1)
         V = cov.covariance(psi, theta, measure)
         assert jnp.linalg.matrix_rank(V) == 1
     #+end_src

** [MED] =ClusteredCovariance= with wildly unbalanced sizes (n_c = 1 vs n_c = N-1)
   - Confirmed gap. Probe runs and produces a finite ~V~, but no test
     verifies behaviour relative to a reference (e.g. balanced
     two-cluster case from =TestTwoClusterFormula=). Cluster-robust
     inference is known to be biased downward when one cluster
     dominates; the framework should at least surface a "max cluster
     fraction" diagnostic or test that the variance does not collapse
     when one cluster is the singleton.
   - Sketch:
     #+begin_src python
     # tests/covariance/test_clustered.py
     def test_unbalanced_clusters_does_not_collapse_to_iid():
         # cluster sizes [1, N-1]; V should differ noticeably from the
         # IID variance unless the within-cluster correlation is zero.
         cov = ClusteredCovariance(cluster_ids=cluster_ids_1_vs_Nm1,
                                   n_clusters=2)
         V_cl = cov.covariance(psi, theta, measure)
         V_iid = IIDCovariance().covariance(psi, theta, measure)
         assert not jnp.allclose(V_cl, V_iid)
     #+end_src

** [MED] Rank-deficient G (a moment is constant in some theta coordinate)
   - Confirmed gap. Probe runs to completion. =theta_hat.b= stays at
     the initial value (b is unidentified) and =Sigma_theta= contains
     ~inf~ on the diagonal for ~b~ and ~nan~ on the off-diagonal. No
     diagnostic flag, no detection. The information matrix
     ~G' V^{-1} G~ is silently rank-deficient and =jnp.linalg.inv= is
     called anyway. Per CLAUDE.md commitment §5 ("Information matrix
     via G' Lambda G, never via numerical Hessian") this matrix is the
     framework's identification gate; it should be rank-checked.
   - Sketch:
     #+begin_src python
     # tests/test_estimator.py
     def test_rank_deficient_G_surfaces_warning_or_diagnostic():
         # Both moments depend on a only; b is unidentified -> G[:, 1] = 0.
         r = estimate(model=psi_b_unidentified, ...)
         # Either: cond(info_matrix) is reported in diagnostics, or
         # raise/flag at the rank-check level.
         assert not jnp.all(jnp.isfinite(r.Sigma_theta.array))  # current behaviour
         # ... preferred: an explicit diagnostic like
         # r.diagnostics.identification_rank == K - 1
     #+end_src

** [MED] =optimistix_lm= with rtol below float64 noise floor
   - Confirmed: probe with =rtol=atol=1e-30= reports
     ~status="converged"~ in 11 iterations for a quadratic. Optimistix
     does not gate on whether the tolerance is achievable; the
     framework accepts any positive float without warning. For a
     numerical-statistics framework that documents "the float32 noise
     floor on a O(0.1)-magnitude whitened residual is around 2e-8"
     in =emu_gmm/__init__.py=, the dual case (rtol below float64
     epsilon) should at least be documented in a test as a known
     edge.
   - Sketch:
     #+begin_src python
     # tests/test_optimizer.py
     def test_optimistix_lm_tolerance_below_float64_eps_converges_or_warns():
         opt = optimistix_lm(rtol=1e-30, atol=1e-30)
         x_opt, info = opt(_quadratic_residual, _X_INIT)
         # Either converged with a documented caveat, or status
         # signals that the tolerance was infeasible.
         assert info.status in ("converged", "max_iterations")
     #+end_src
