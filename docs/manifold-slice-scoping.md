## Build spec

# Minimal-Slice Build Spec — #12 Manifold Epic, scoped to unblock the K-Aggregators port

## Objective (the one acceptance that defines "done")

Make `Product(PSDFixedRank(5, K), Euclidean(1))` **estimable end-to-end**:
`estimate()` recovers a known `Gamma = A @ A.T` (rank K, 5x5 PSD) and scalar
`phi` from synthetic data, returns a finite gauge-aware `Sigma_theta`, a
correctly-calibrated `J_stat`, and a `result.theta.components() -> (A, phi)`
readout. Reuse the **landed Positive(1,1) slice wiring** (`_resolve_optimizer`
dispatch, `RiemannianLM` skeleton, `ManifoldSpec`/`LeafSpec`, J_dof arithmetic)
and the **landed manifold TYPES** (`PSDFixedRank`, `Product`,
`euclidean.py`/`positive.py` operators) unchanged where possible.

## Scale: ~7 PRs across 4 critical-path phases + 1 readout phase + 1 validation phase

This is genuinely a **6-phase, ~7-PR** slice, not a one-PR relaxation. The four
critical-path walls (verified in src) are independent code surfaces that must
all land before a single `estimate()` call runs, and each has a wrong-but-silent
failure if landed half-way (see top_risks). The non-negotiable additions
(`ManifoldLeaf` PyTree node, manifold-aware 3-tuple flatten, non-scalar spec
builder, per-leaf solver retraction/projection + gauge lambda-floor,
`pinv_eigvalrule`, per-leaf G_riem assembly, `components()` accessor) are each
~50-150 LOC plus tests. Phases 5-6 (readout, validation) are smaller but are
where the K-Agg contract is actually satisfied.

## Phase 0 — Pin the model-boundary + backend contract (docs/decisions only, ~0.5 PR, may fold into Phase 1)

No code path consumes a `Product` object today; the estimable surface is the
per-field `__emu_manifolds__` class-attr on scalar leaves (params.py:203-205).
**Decision to pin before writing flatten code:** the v2 non-scalar param is
expressed as a `ManifoldLeaf`-wrapped pytree (matrix/vector leaf carrying its
own `ManifoldParam`), OR as a `__emu_manifolds__` annotation pointing a
non-scalar field at `PSDFixedRank(5,K)` / `Euclidean(1)`. The K-Agg caller wants
`theta` delivered to the model as a structured tuple `(A, phi)` and read back via
`result.theta.components()`. Pin: native `Product`/`PSDFixedRank` constructors are
the supported surface; **`from_pymanopt` is NOT provided** in this slice (see
open questions / INT-10). Document this in the manifolds package docstring so the
consumer rewrites `Manifold.from_pymanopt(...)` to native constructors.

Touches: `src/emu_gmm/manifolds/__init__.py` docstring; a short note (no new
`docs/mcar-asymptotics-riemannian.org` — deferred).

## Phase 1 — Manifold-aware flatten core (CRITICAL PATH #1, ~1.5 PRs)

The single first wall. `flatten_params` returns the v1 scalar-only 2-tuple and
raises on any `ndim != 0` leaf (params.py:60-64); `unflatten_params` re-indexes
flat scalars (params.py:106). A `(5,K)` `A` block cannot survive the round-trip.

**Build:**
1. `ManifoldLeaf` opaque PyTree node (`register_pytree_node_class`) carrying the
   leaf array + its `ManifoldParam`. Lives in `manifolds/` (new file
   `manifold_leaf.py`) since `ManifoldLeaf` is referenced only in doc comments
   today (grep confirms absent in src).
2. Manifold-aware `flatten_params(params) -> (flat, treedef, manifold_spec)`
   3-tuple that concatenates each leaf's ambient block into a `total_ambient_dim`
   buffer (length `5*K + 1` for the target). **Preserve the v1 2-tuple call
   bitwise** for all-scalar trees — either a `flatten_params_v2` companion or a
   keyword-gated return so the 226 v1 tests and every existing
   `flatten_params(...)` caller (estimator.py:323,564,615; types.py:427;
   riemannian_lm.py:75) keep working. The cleanest is: keep `flatten_params`
   2-tuple, add `flatten_params_with_spec` 3-tuple; migrate the v2 call sites
   only.
3. `unflatten_params` that slices `flat[offset:offset+prod(ambient_shape)]`,
   reshapes to `ambient_shape`, and reconstructs the pytree.

Touches: `src/emu_gmm/_internal/params.py:31-149`,
`src/emu_gmm/manifolds/manifold_leaf.py` (new).
Acceptance: round-trip `flatten -> unflatten` of `(A:(5,K), phi:())` preserves
each leaf shape exactly.

## Phase 2 — Non-scalar spec builder (CRITICAL PATH #2, ~1 PR)

`manifold_spec_from_params` is the ONLY spec builder `estimate()` calls
(estimator.py:257) and it hard-raises on `arr.ndim != 0` (params.py:213-218),
setting `ambient_shape=()` and `offset += manifold.dimension` unconditionally.

**Build:** make it accept non-scalar leaves (via `ManifoldLeaf` or non-scalar
`__emu_manifolds__` annotation), populating each `LeafSpec.offset` and
`ambient_shape` from the **actual array shape** with `offset += prod(ambient_shape)`
(NOT `manifold.dimension`). Sum each `leaf.manifold.gauge_dim` into
`total_gauge_dim`. `LeafSpec.ambient_shape` already exists (spec.py:43-46).

**Guard against INT-06 (silent block-boundary corruption):** assert
`sum(prod(ls.ambient_shape) for ls) == len(flat) == total_ambient_dim` and each
`LeafSpec.ambient_shape` matches its leaf. For the target spec:
`total_ambient_dim == total_dimension == 5*K+1`, `total_gauge_dim == K(K-1)/2`.

Touches: `src/emu_gmm/_internal/params.py:151-238`.
Note: `_resolve_optimizer` (estimator.py:159-189) is ALREADY correct and will
route this non-Euclidean spec to `riemannian_lm`/v2 with **zero dispatch
changes** — verified. No work there.

## Phase 3 — Per-leaf solver: ambient-block retraction/projection + gauge lambda-floor (CRITICAL PATH #3, ~1.5 PRs)

`_RiemannianLM` documents "native scalar leaves... no Product machinery" and
assumes "offset == leaf index" (riemannian_lm.py:79-82). It builds a per-
*coordinate* `is_pos`, retracts elementwise `x*exp(d/x)` vs `x+d`
(riemannian_lm.py:105-107), and solves a plain Gram `(A + lam*I)d = -g` with
only a `1e-12` floor (riemannian_lm.py:150-157) — no gauge floor, no projection.

**Build (reusing the existing `jax.lax.while_loop` + jacfwd skeleton):**
1. Iterate per `LeafSpec`: slice the `(M, offset:offset+size)` column block, reshape
   step blocks to `ambient_shape`, dispatch to the leaf manifold's `retraction`
   (PSDFixedRank `Y+V`, psd_fixed_rank.py:114-123) and `projection` (horizontal
   Lyapunov solve, psd_fixed_rank.py:93-112) — both already implemented and
   unit-tested. The embedded PSD metric coincides with ambient Frobenius
   (psd_fixed_rank.py:147-150), so a plain ambient lstsq is correct for the
   horizontal part; `inner_product` is NOT strictly required.
2. Add `lam_floor = gauge_floor * total_gauge_dim` so the Gram stays PD along the
   `K(K-1)/2` gauge directions in a controlled way (R2/R6/INT-07). Without this
   the iterate wanders the O(K) fibre and never certifies.
3. Apply per-leaf `projection` to the step (or Jacobian columns) so the step has
   no arbitrary gauge component.

**Also address INT-08 (status="traced" == converged hazard):** the `done` flag
is already a traced bool in the carry (riemannian_lm.py:185-186). Propagate it
out so the estimator can flag `steps == max_steps` / large `final_gradient_norm`
as non-convergence, rather than collapsing to `status=="traced"` always-converged
(estimator.py:690).

Touches: `src/emu_gmm/manifolds/riemannian_lm.py:75-206`.

## Phase 4 — Gauge-aware Sigma_theta: pinv_eigvalrule + per-leaf G_riem (CRITICAL PATH #4, ~1.5 PRs)

Two coupled bugs in the inference block:
- `G_riem` is built by `range(K)` over flat scalar columns calling
  `retraction_differential(theta_flat[j])` per scalar (estimator.py:503-512).
  With K = **field count** (2), this silently DROPS columns 2..(5K) of the
  `(M, 5K+1)` Jacobian (INT-01, INT-02) → a 2x2 info matrix that is NOT the
  information matrix.
- `Sigma_local = jnp.linalg.inv(info_local)` (estimator.py:515) is singular for a
  quotient: `info` is rank-deficient by `gauge_dim = K(K-1)/2` → inf/nan (R3,
  INT-03).

**Build:**
1. Replace every dimension-keyed `K` in the inference/label path with
   `manifold_spec.total_dimension` (ambient tangent dim). Re-derive the
   under-identification guard (estimator.py:289) as
   `total_dimension - total_gauge_dim > M`. (INT-01.) The v1 invariant holds:
   for scalar trees `total_dimension == K == field count`, so the 226 tests pass.
2. Rebuild `G_riem` to iterate over `leaf_specs`: slice `G_local[:, offset:offset+size]`,
   reshape to `ambient_shape`, apply the leaf's horizontal `projection`
   (PSDFixedRank) / identity (Euclidean) as an `(M, leaf_dimension)` block, then
   hconcat to an `(M, total_dimension)` matrix. (INT-02.)
3. Implement `pinv_eigvalrule(info, drop_smallest=manifold_spec.total_gauge_dim)`
   (absent in src, grep-confirmed) — drop the `gauge_dim` SMALLEST eigenvalues
   **by COUNT** (static int, vmap/jit-safe per ManifoldGMM PR #33), NOT by
   magnitude threshold. Replace `inv()`. For `total_gauge_dim==0` (v1/Positive)
   it reduces to `inv()` → bitwise v1 non-regression.

**Sigma_theta gauge convention (minimal):** report the **ambient natural-scale
sandwich** with the `gauge_dim` exact gauge zeros pinned out — i.e.
`pinv_eigvalrule` of the horizontal-projected `(G'ΛG)`. The
`retraction_differential==1` Convention-B scaling (estimator.py:490-512) stays
the identity, matching the landed Positive slice. **DEFER** the full gauge-aware
pull-back to the Gamma=YY^T / eigenvalue functionals (delta method, R1/R5) and
the `SigmaThetaBasis` dataclass — but DO surface `gauge_nullspace_dim` (=
`spec.total_gauge_dim`, trivially available) in Diagnostics so users distinguish
gauge zeros from numerical noise. J_dof is ALREADY correct arithmetic
(estimator.py:436-438); it now calibrates because `total_gauge_dim` is populated
(R7/INT-09).

Touches: `src/emu_gmm/estimator.py:289,436-438,503-515,639-644`; new
`pinv_eigvalrule` helper (in `_internal/` or `inference/`).

## Phase 5 — components() readout + result-path flatten (READOUT, ~1 PR)

The K-Agg structural contract: `result.theta.components() -> (A, phi)`, callers
compute `Gamma_hat = A @ A.T`, `theta = exp(phi)`, `eigvalsh(Gamma)`; warm-starts
read `prev.theta.components()`. Today `result.theta_hat` is a bare pytree (no
`.components()`), and `coef_table` re-calls `flatten_params` (types.py:427) which
raises on non-scalar leaves (INT-04, INT-05).

**Build:**
1. A `components()` accessor on the result's theta (a thin `ManifoldPoint`-like
   wrapper, or a `result.components()` method that returns the unflattened
   per-leaf tuple in field order). Optionally `.gamma` (=`A@A.T`) and
   `.diagonalized()` since manifold_tools.py:55-77 reads them — but `.gamma` /
   `.diagonalized()` are downstream conveniences the caller can compute from
   `components()`; **flag as optional**.
2. Route `coef_table` / `standard_errors` / the final `_run` flatten through the
   manifold-aware flatten so a user inspecting results doesn't hit the
   `ndim != 0` raise. Either build `coef_table`'s estimate column from the same
   ambient flatten the SE axis uses, or define the table over tangent-basis labels.

Touches: `src/emu_gmm/types.py:382-427` (`coef_table`, `standard_errors`,
`EstimationResult`), the v2 flatten call sites.

## What this slice EXPLICITLY does NOT do (deferred)

- **Trust-region solver** — RiemannianLM suffices per kaggNeed; #9 TrustRegions deferred.
- **Full gauge-aware Sigma_theta pull-back** to Gamma=YY^T / eigenvalue
  functionals (delta method through the retraction; the gauge-INVARIANT
  downstream quantity) — issue #42. Minimal slice reports the `pinv_eigvalrule`
  ambient-horizontal sandwich + `gauge_nullspace_dim` only.
- **`SigmaThetaBasis` dataclass, `Sigma_theta_ambient()`, `spectral_gap`
  diagnostic, `_internal/labels.tangent_basis_names` aggregator** — Sigma_theta
  axes get positional tangent labels, NOT mislabelled field-names (INT-12/R5).
- **pymanopt backend** (`from_pymanopt`, `OptimistixManifoldAdapter`) — native
  constructors only; issue #3.
- **Demographics leaf** `Euclidean(n_demog_chars)` — the general N-leaf Product
  subsumes it for free; the K-Agg primary (no-demographics) path is the target.
- **`docs/mcar-asymptotics-riemannian.org`** companion note — needed for
  review/correctness sign-off, off the running-code critical path.
- **Reverse-mode AD through the solver** — NOT required and NOT built; AD is on
  the moment function at theta_hat ONLY (commitment 5 / kaggNeed / R4).
- **k/n > 0.7 spectral-gap regime** — K-Agg K=2,3 / n=5 stays at k/n=0.4-0.6;
  surface the caveat as a diagnostic, do not support the high-gauge-fraction case.


## Acceptance (the must-pass gate)

- **Synthetic Gamma+phi recovery (the gate)** — estimate() on Product(PSDFixedRank(5,2), Euclidean(1)) and PSDFixedRank(5,3) over synthetic data with known A (Gamma=A@A.T) and phi recovers Gamma_hat ~= A@A.T and phi_hat ~= phi to the acceptance tolerance; result.converged is True with small final_gradient_norm (NOT merely status=='traced'). Recovery is asserted on the gauge-INVARIANT functionals Gamma_hat and eigvalsh(Gamma_hat), never on raw Y entries.
- **ManifoldGMM cross-check on the quotient** — Gamma_hat and J_stat from emu-gmm riemannian_lm agree with the ManifoldGMM/pymanopt-TrustRegions baseline on the same cereal-style DGP to within tolerance; agreement is checked on functionals of Gamma (gauge-invariant) and on J, NOT on raw Y_hat (which differs by an O(K) gauge rotation).
- **Gauge invariance of point estimate** — Two starts Y0 and Y0@Q for a random Q in O(K) return the same Gamma_hat, J_stat, and Sigma_theta(Gamma) within tolerance; the solver does not wander the O(K) fibre (gauge lambda-floor + per-leaf horizontal projection are exercised).
- **Finite gauge-aware Sigma_theta** — Sigma_theta is finite (no inf/nan) for the PSDFixedRank leaf; pinv_eigvalrule drops EXACTLY total_gauge_dim==K(K-1)/2 smallest eigenvalues of info_matrix by COUNT; standard_errors and coef_table return without raising; info_matrix.shape == (total_dimension, total_dimension) == (5K+1, 5K+1) and G_riem uses ALL ambient columns (no silent column drop).
- **J_dof calibration from a populated gauge_dim** — J_dof == M - (total_dimension - total_gauge_dim) == M - (5K+1 - K(K-1)/2) with total_gauge_dim flowing from the spec builder (NOT hard-coded 0); a deliberately gauge-VIOLATING residual (depends on Y not through YY^T) is rejected or shows the gauge directions are not null.
- **components() / structural readout contract** — result.theta.components() returns the per-leaf tuple (A, phi) in field order; callers can compute Gamma_hat=A@A.T, theta=exp(phi), eigvalsh(Gamma); a warm-start reading prev.theta.components() round-trips into a new estimate().
- **flatten round-trip preserves leaf shapes** — manifold-aware flatten_params -> unflatten_params of a Product param (A:(5,K), phi:()) reconstructs each leaf with its exact shape; sum of LeafSpec block widths == flat buffer length == total_ambient_dim == 5K+1; each LeafSpec.ambient_shape matches its leaf.
- **v1 bitwise non-regression** — All 226 v1 tests pass unchanged; for all-Euclidean/scalar-Positive trees total_gauge_dim==0 so pinv_eigvalrule reduces to inv(), total_dimension==K==field-count, and the Positive(1,1) slice (test_estimator_positive.py: Sigma_theta == plain Euclidean estimate) is preserved bitwise.

## Open questions (3 resolved; remainder carry recommendations)

1. Solver: ship ONLY the Riemannian LM with gauge lambda-floor, or also build the Riemannian TrustRegions (#9)? kaggNeed says EITHER suffices and LM is the smaller surface; recommend LM-only for this slice, but confirm the K-Agg owner accepts LM-vs-TR equivalence verified ON THE QUOTIENT (functionals of Gamma) rather than on raw Y.

2. Reverse-mode AD scope (#77): confirm the inference G_riem is built by AD of the moment function at the FIXED theta_hat (delta method, commitment-5 pattern), and that differentiating through riemannian_lm's lax.while_loop is explicitly OUT of scope. The reference (gmm_inference.py) treats the solver as a black box; is anyone downstream expecting differentiable-through-solver behavior?

3. Sigma_theta basis (#42): for THIS slice, is the pinv_eigvalrule ambient-horizontal sandwich + reported gauge_nullspace_dim sufficient, or does K-Agg need the gauge-INVARIANT pull-back to Gamma=YY^T / eigenvalue functionals (delta method through the retraction) before they can use the SEs? The raw per-entry Y SEs are gauge-arbitrary; only functionals of Gamma have meaning. This is the biggest scope fork.

4. Backend (#3): ship native Product/PSDFixedRank ONLY (consumer rewrites Manifold.from_pymanopt to native constructors), or add a thin from_pymanopt adapter? Adding pymanopt as a runtime dep needs explicit user sign-off (CLAUDE.md 'don't surprise the user'). Recommend native-only + an OPTIONAL gated parity test against pymanopt.

5. Param expression: is the v2 non-scalar leaf a ManifoldLeaf-wrapped pytree, or a __emu_manifolds__ class-attr pointing a non-scalar field at PSDFixedRank(5,K)? The former is what the plan's flatten core assumes; the latter reuses the landed Positive-annotation mechanism but needs the spec builder to read shape from the field array. Pick one before Phase 1.

6. Rank K selection / CUE vs Identity: K is the user-supplied rank (runs use K=2,3) — is auto rank-selection (e.g. by eigenvalue gap of Gamma) in scope, or strictly user-supplied? And confirm CUE-default + Identity-optional weighting (already in v1) needs no new work for the manifold path.

7. status=='traced' hazard (#78 generalised): is fixing the 'converged while traced' collapse (propagating the done flag out of the while_loop) in scope for THIS slice, or deferred? It is benign for scalar Positive but dangerous for a wandering gauge orbit; recommend fixing it here as cheap insurance.


## Top risks

- **[critical] Silent column-drop: K (field count) used where total_dimension (5K+1) is needed, producing a wrong-but-finite Sigma_theta with no error** — Replace every dimension-keyed K in the inference/label/guard path (G_riem range(K), info shape, params axes, K>M guard at estimator.py:289) with manifold_spec.total_dimension; rebuild G_riem to iterate leaf_specs over the full (M, total_dimension) Jacobian. Regression test asserting info_local.shape == (5K+1, 5K+1) and that G_riem reads all ambient columns. v1 invariant total_dimension==K==field-count keeps 226 tests green.
- **[critical] Plain inv() of a structurally rank-deficient (gauge) info matrix -> inf/nan (or huge finite garbage that mimics weak identification)** — Implement pinv_eigvalrule(info, drop_smallest=total_gauge_dim) dropping K(K-1)/2 smallest eigenvalues BY COUNT (static int, vmap-safe per ManifoldGMM PR#33), replacing inv(); reduces to inv() when total_gauge_dim==0. Test that exactly total_gauge_dim eigenvalues are dropped and Sigma_theta is finite. Surface gauge_nullspace_dim and (deferred) spectral_gap so a near-zero IDENTIFIED eigenvalue is not conflated with a gauge zero.
- **[critical] Relaxing the manifold_spec_from_params raise without fixing offset/ambient_shape -> internally inconsistent block boundaries -> silent data corruption ('looks like it ran')** — Spec builder must set LeafSpec.offset and ambient_shape from the actual array shape with offset += prod(ambient_shape) (NOT manifold.dimension), and sum each leaf.manifold.gauge_dim into total_gauge_dim. Assert sum of block widths == flat length == total_ambient_dim and each ambient_shape matches its leaf. This is the most likely silent failure of a naive build.
- **[high] Solver wanders the O(K) gauge fibre: plain Euclidean Gram has no gauge lambda-floor and no per-leaf horizontal projection, so LM never certifies / converges to an arbitrary Y** — In riemannian_lm iterate per LeafSpec (reshape ambient blocks, dispatch to PSDFixedRank.retraction + horizontal projection — both already implemented/tested), and add lam_floor = gauge_floor*total_gauge_dim so the Gram stays PD along the K(K-1)/2 gauge directions. Verify equivalence with ManifoldGMM ON THE QUOTIENT (Gamma, J), never on raw Y.
- **[high] Result-path break: coef_table/standard_errors/final unflatten re-call the v1 scalar-only flatten which raises on non-scalar leaves, AND result.theta has no components() accessor** — Land the manifold-aware flatten and route coef_table/standard_errors/_run through it; add result.theta.components() returning the per-leaf (A, phi) tuple. Test flatten/unflatten round-trip of a Product param and a warm-start reading prev.theta.components(). Until landed, document that only scalar-leaf trees are estimable to avoid a half-working structured path.
- **[medium] J-statistic mis-calibration if total_gauge_dim is mis-populated (forgotten or double-counted under Product): J_dof silently shifts by K(K-1)/2, mis-sizing the chi2 test with no error** — J_dof = M - (total_dimension - total_gauge_dim) arithmetic is already correct (estimator.py:436-438); the untested input is total_gauge_dim. Assert J_dof == M - (5K+1 - K(K-1)/2) and add the gauge-invariance contract test (deliberately gauge-violating residual) so a gauge leak into dof is caught. Cross-check total_gauge_dim flows from the spec builder, not a hard-coded 0.
- **[medium] Sigma_theta reported in raw ambient (n*k) Y-coordinates whose per-entry SEs are gauge-arbitrary (only Gamma=YY^T functionals have meaning); axis labels may also mislabel them as field-names** — For THIS slice report the pinv_eigvalrule ambient-horizontal sandwich with positional tangent labels (NOT mismatched field-names) and surface gauge_nullspace_dim; explicitly DEFER (open question #3) the gauge-invariant delta-method pull-back to Gamma/eigenvalues and the SigmaThetaBasis/tangent_basis_names aggregator to issue #42. Document that raw per-entry Y SEs are not interpretable so no one reads an SE off A_t_01.
